NAME
gcloud alpha bigquery copy - (ALPHA) Copy one table to another
SYNOPSIS
gcloud alpha bigquery copy SOURCE DESTINATION [--async] [--fingerprint-job-id] [--format FORMAT] [--help] [--if-exists IF_EXISTS; default="prompt"] [--job-id JOB_ID] [--log-http] [--project PROJECT_ID] [--quiet, -q] [--status STATUS; default="periodic"] [--trace-token TRACE_TOKEN] [-h]
DESCRIPTION
(ALPHA) If the table does not exist, it is created. Otherwise, use --if-exist flag to choose desired behaviour.
POSITIONAL ARGUMENTS
SOURCE
the table to be copied from
DESTINATION
the table to be copied to
FLAGS
--async
If True, create an asynchronous job, and use the success of job creation as the error code. If False, wait for command completion before returning, and use the job completion status for error codes.
--if-exists IF_EXISTS; default="prompt"
What to do if the destination table already exists.
--job-id JOB_ID
A unique job ID to use for the request.
--status STATUS; default="periodic"
Whether the status of the copying job should be reported periodically, every time the status changes, or not at all.
GLOBAL FLAGS
--fingerprint-job-id
Whether to use a job id that is derived from a fingerprint of the job configuration.
--format FORMAT
Specify a format for printed output. By default, a command-specific human-friendly output format is used. Setting this flag to one of the available options will serialize the result of the command in the chosen format and print it to stdout. Supported formats are: json, text, yaml.
--help
Display detailed help.
--log-http
Logs all HTTP server requests and responses to stderr.
--project PROJECT_ID
The Google Cloud Platform project name to use for this invocation. If omitted then the current project is assumed.
--quiet, -q
Disable all interactive prompts when running gcloud commands. If input is required, defaults will be used, or an error will be raised.
--trace-token TRACE_TOKEN
Token used to route traces of service requests for investigation of issues.
-h
Print a summary help and exit.
EXAMPLES
To copy table from projectX to current project in datasetY:
$ gcloud alpha bigquery copy projectX/datasetX/tableX \
    datasetY/tableY
NOTES
This command is in the Google Cloud SDK bigquery component. See installing components if it is not installed.
This command is currently in ALPHA and may change without notice.
